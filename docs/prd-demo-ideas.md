# AI Demo Idea Request PRD

## Overview

The Demo Ideas page (`/demo-ideas`) allows prospects to submit use case ideas for their personalized AI demo. It provides AI-generated industry-relevant suggestions and a form to submit 1-3 custom ideas.

## Engagement Goals

**Primary THINK:** Impact, Unlock
**Primary FEEL:** Inspired, Enabled

### Goal Implementation

| Goal | How to Achieve |
|------|----------------|
| **Impact** | Ideas become reality; "tell us your challenge, we'll build it live" |
| **Unlock** | AI suggestions reveal possibilities they hadn't considered |
| **Inspired** | AI-generated suggestions show product intelligence; "this knows my industry" |
| **Enabled** | Simple form, clear outcome; "submit ideas, see them built" |

### Key Messaging
- Header: "What should we build for you?"
- Subtext: "Submit your ideas and watch them come to life in your demo"
- AI Suggestions: "Based on companies like yours, here's what works"
- Form: "Tell us your toughest internal tool challenge"

### Inspiration Triggers
- AI suggestions demonstrate industry knowledge
- Show complexity estimates = "yes, we can build that"
- Related use cases expand thinking
- "Companies like yours built X" examples

### Enablement Design
- Pre-filled suggestions reduce friction
- Priority field helps AE prep appropriately
- Estimated demo time sets expectations
- Immediate confirmation with next steps

### Impact Framing
- Every idea submitted = voice in their demo
- "Your AE will review and prepare to build these live"
- Follow-up shows ideas were taken seriously
- Points reward reinforces value of contribution

## Features

### 1. AI-Generated Use Case Suggestions

Display industry and role-specific use cases generated by AI:

```typescript
interface UseCaseSuggestion {
  id: string;
  title: string;
  description: string;
  industry: string;
  role: string;
  complexity: 'simple' | 'medium' | 'complex';
  estimatedDemoTime: string;
  tags: string[];
  selected?: boolean;
}
```

Generation based on:
- Prospect's industry (from CustomerContext)
- Roles of people in the meeting
- Company size and tech stack
- Known pain points (from AI Research)

### 2. Use Case Card Component

Each suggestion card shows:
- Use case title
- Brief description
- Complexity badge
- Estimated demo time
- Relevant tags
- "Select for demo" checkbox
- "Customize this idea" button

### 3. Custom Idea Submission Form

Allow prospects to submit their own ideas:

```typescript
interface DemoIdeaSubmission {
  id: string;
  title: string;
  description: string;
  priority: 'high' | 'medium' | 'low';
  expectedOutcome: string;
  additionalContext?: string;
  attachments?: string[];
  submittedAt: string;
}
```

Form fields:
- Idea title (required)
- Description (required, textarea)
- Priority level (dropdown)
- Expected outcome (what success looks like)
- Additional context (optional)
- File attachments (optional, for mockups/specs)

### 4. Idea Limit & Validation

- Minimum: 1 idea required
- Maximum: 3 ideas allowed
- At least one must be "high" or "medium" priority
- Character limits: Title (100), Description (500)

### 5. AI Enhancement

When user submits an idea, AI can:
- Suggest refinements
- Add relevant technical details
- Estimate complexity
- Identify prerequisites

```typescript
interface AIEnhancedIdea extends DemoIdeaSubmission {
  aiSuggestions: {
    refinedDescription?: string;
    technicalNotes?: string;
    estimatedComplexity: string;
    prerequisites: string[];
    relatedUseCases: string[];
  };
}
```

### 6. Submission Confirmation

After submission:
- Success animation
- Summary of submitted ideas
- "Your AE will review these before the call"
- Edit/modify option until call date
- Points awarded notification

## Rewards Integration

Points awarded:
- 10 points: Open Demo Ideas page
- 15 points: Select an AI suggestion
- 25 points: Submit custom idea
- 50 points: Submit all 3 ideas (bonus)

## Animations

1. **Page entry:** BlurFade on sections
2. **Card selection:** Scale + border highlight
3. **Form add:** Slide down animation
4. **Submit:** Success confetti
5. **AI suggestions:** Staggered reveal

## Integration

- CustomerContext for industry/role data
- ProgressContext for tracking completion
- RewardsContext for points
- AI Research data for pain points context

## Responsive Behavior

- Desktop: 3-column suggestion grid, form to the right
- Tablet: 2-column grid, form below
- Mobile: Single column, stacked layout

## Validation States

### Empty State
- Show AI suggestions prominently
- "Select or create your demo ideas"
- Minimum 1 idea required message

### Partial State
- Show selected/submitted count
- "Add more ideas" prompt
- Points preview

### Complete State
- All 3 ideas submitted
- "You're all set!" message
- Edit option available
- Bonus points awarded

## Error Handling

- Network error: Save draft locally
- Validation error: Inline field messages
- AI generation error: Show cached/mock suggestions
- Submission error: Retry with saved data
